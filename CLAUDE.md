## 6 · Conventions (these are non-negotiable)

### 6.1 · Tickets — `SILK-NNNN`

Every open work item in PROGRESS.md carries a `SILK-NNNN` ID. Spec: [docs/TRACKING_CONVENTION.md](docs/TRACKING_CONVENTION.md). Next available: `./scripts/next-ticket-id.sh`.

**Commit format:** `<type>(<scope>): SILK-NNNN — <subject>`

```
feat(auth): SILK-NNNN — OTP verify endpoint
fix(mobile): SILK-NNNN, SILK-NNNN — confirm password validator + locale strings
refactor(infra): SILK-NNNN — collapse interceptors
docs: SILK-NNNN — log Apple Sign In blocker
```

> In docs always use `SILK-NNNN` placeholder. Real numeric IDs in docs get picked
> up by `./scripts/next-ticket-id.sh` as burned allocations.

Trivial changes (typos, single-file lint, comments) don't need a ticket — just commit.

### 6.2 · Production-quality always

From user memory ([feedback-production-quality-mandate]): **never accept MVP, hardcoding, or "sodda yechim"**. Everything dynamic via admin panel — no hardcoded URLs, copy, prices, feature flags. If you find yourself writing `if env == "dev"`, stop and check if it belongs in `system_settings` or `feature_flags`.

### 6.3 · Clean Architecture (ADR-0003)

Three layers, dependency arrow points inward:

```
api (routers)  →  domain (entities + protocols)  ←  infrastructure (impls)
```

- **Domain layer** has zero framework imports — pure dataclasses + protocols
- **Infrastructure** satisfies domain protocols structurally (no inheritance)
- **API** wires the two together inside router functions; never imports infrastructure from domain
- Flutter mirrors this: `domain/` has zero Flutter imports; `data/` implements; `presentation/` consumes

### 6.4 · Multi-tenant + residency partitioning

- Every user-owned table has a `residency_region` column partitioned into `global` and `eu` (Postgres declarative partitioning)
- Repositories take `residency_region` as a parameter — never assume `global`
- `tenant_id` is also required on most rows (RBAC + branding)
- See `docs/architecture/02-identity-rbac-gdpr.md`

### 6.5 · UUIDv7 primary keys (ADR-0004)

Use `app.uuidv7()` SQL function for new IDs. Never `uuid_generate_v4()`. v7 IDs are time-ordered → better B-tree locality.

### 6.6 · Audit chain HMAC

Sensitive writes call `app.audit(actor_id, action, payload)` which appends to `audit_log` with HMAC chain (each row signs the previous). `SILKLENS_AUDIT_HMAC_KEY` must NEVER be regenerated in prod (would break chain verification).

## 8 · Gotchas / footguns (read before you break things)

### 8.1 · mail.ru blocks HTML emails from `onboarding@resend.dev`

We send **plain-text only** OTP emails. Resend dashboard shows "Blocked due to content" for HTML payloads from the shared sender. Do NOT re-add an `html=` body until SILK-0010 (`silklens.app` domain verify) is closed.

See: `services/api/src/api/routers/auth.py` `_otp_text()`.

### 8.2 · Settings are `@lru_cache`'d — `.env` changes require API restart

`src/core/settings.py` uses `@lru_cache(maxsize=1)`. Editing `.env` does NOT hot-reload. Kill `python -m src` and restart.

### 8.3 · Alembic migrations are timestamp-ordered and never reorder

Filename pattern: `YYYYMMDD_NNNN_<name>.py`. Sequence is contiguous (0001…0092 today). Adding a new migration → use `make api-revision m="…"` to get the next number; do NOT hand-edit filenames.

Round-trip test before every commit that touches schema:
```bash
make api-downgrade && make api-migrate && make api-test
```

### 8.4 · MIUI (Xiaomi) silently drops adb tap events outside button bounds

When automating UI tests on the Redmi device (Physical 720x1640), get exact coords via `uiautomator dump` first. Taps **just outside** a clickable bound are accepted at the input layer but the gesture detector ignores them, so you see no error and no state change. Real input device: `omnivision_tcm_touch` (event4).

### 8.5 · TextField focus and adb `input text`

Each `adb shell input text` writes to **currently focused field**. After typing, hide the keyboard (`KEYCODE_BACK`) before tapping the next field — keyboard occlusion shifts targets. Re-dump `uiautomator` between steps.

### 8.6 · Residency partitioning bites silently

`SELECT * FROM users WHERE id = …` against the parent table returns nothing if you forget the `residency_region` predicate. Always query through the repository which adds it.

### 8.7 · `apps/mobile/android/app/google-services.json` + `client_secret_*.json` are real secrets

Listed in `.gitignore`. NEVER `git add` them manually. If you see them in `git status`, something is wrong.

### 8.8 · Rate-limit lockout (SEC-005)

5+ failed `/login` attempts in 10 min from same IP → 423 LOCKED for 15 min. While debugging, clear with:
```bash
docker exec silklens-postgres psql -U silklens silklens -c "DELETE FROM login_attempts WHERE identifier LIKE '%test%';"
```

## 12 · Quick mental model

Think of SilkLens as **two systems**:

1. **Backend FastAPI** — bounded contexts (18 domains) over a residency-partitioned, RBAC-gated, audit-chained Postgres. Each context = domain + infrastructure + router. Event-driven via `event_outbox`.

2. **Flutter app** — Clean Architecture with Riverpod state, GoRouter navigation, Dio HTTP, Isar local cache, BackdropFilter glass UI on a 4-variant theme system. Bilingual+ (uz/ru/en/zh) baked in.

Everything else (admin panel, AI server, media pipeline) hangs off these two.
